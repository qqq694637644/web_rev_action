# Current-site inventory checklist

Use this checklist before replay or mutation. Record each item as observed, missing, ambiguous, or not applicable.

## Page and UI

- current URL, title, page ID, and visible account state;
- primary controls and state-changing actions;
- page transitions, dialogs, loading states, and persistent results;
- whether state survives reload or navigation;
- visible error and validation behavior.

## Ordinary network

For each relevant request, record:

```text
exact evidence or observation ID
method
origin and path
query parameter names
request Content-Type and bounded shape
HTTP status
response Content-Type
initiator and script source
association confidence
```

Do not treat URL similarity as exact association.

## Streaming and long-lived transport

Record independently:

```text
transport observation: fetch/XHR, EventSource, WebSocket, worker, unknown
HTTP status when observed
request lifecycle status
raw and semantic event counts
primary event source
terminal reason and matched condition
truncation or parser gaps
artifact completeness
```

Lifecycle values such as `finished`, `canceled`, or `failed` are not HTTP status codes.

## Workers, storage, and authentication

Look for evidence of:

- service workers, dedicated workers, or shared workers when exposed by current evidence;
- IndexedDB, localStorage, sessionStorage, Cache Storage, or cookies when exposed by current evidence;
- browser-managed versus script-provided authentication state;
- CSRF or request-context inputs;
- response values that feed later requests.

Absence of evidence is not evidence of absence. Current public Browser operations do not directly expose every worker target, WebSocket frame, storage API, or paused runtime stack. When the available evidence cannot observe one of these dimensions, record the capability gap instead of claiming it was fully inspected.

## Dynamic identifiers

Classify each dynamic value by source:

```text
generated by page code
preserved from source request
extracted from a prior response
provided as literal or manual experiment input
unknown
```

Record case sensitivity, multiplicity, order, and raw query encoding when relevant.

## Evidence gaps

Before proposing a replay, list the exact gap it will address. Common gaps:

- request association is ambiguous;
- source response body or headers are incomplete;
- stream termination is unknown;
- initiator source is unavailable;
- dynamic input provenance is unknown;
- persistent state has not been verified;
- account or page context differs across observations.
